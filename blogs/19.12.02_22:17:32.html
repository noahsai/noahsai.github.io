debian下aria2配置，我用于浏览器插件的

19.12.02_22:17:32

1.安装aria2

sudo apt install aria2


2.创建aria2.conf ，位置其实可以随便，只要启动aria2时修改.conf文件路径即可，不一定要在/etc/aria2里面

先再/etc下创建aria2文件夹

sudo mkdir /etc/aria2

然后创建创建/etc/aria2/aria2.conf

我用gedit创建及编辑的

sudo gedit /etc/aria2/aria2.conf

然后百度里面的内容，根据自己需要修改模板即可

我用的：
dir=/home/用户名/下载/aria
disable-ipv6=true
enable-rpc=true
rpc-allow-origin-all=true
rpc-listen-all=true
rpc-listen-port=6801
rpc-secret=***
continue=true
input-file=/etc/aria2/aria2.session
save-session=/etc/aria2/aria2.session
max-concurrent-downloads=5
max-connection-per-server=16


以上内容说明：

dir为下载路径，

rpc-secret=***浏览器插件需要设置用于验证的

input-file和save-session需要指向一个空的aria.session文件，位置自己决定，要手动创建，空的即可

max-connection-per-server这个选项为单个服务器可连接的线程数，最大16,通常越多越快，我习惯直接16

rpc-listen-port=需要注意，如果你用终端启动aria2时提示端口已占用时可以修改这里，例如我的6800已经占用，所以我设为6801


保存好aria2.conf后创建上面说的aria.session

空的即可，位置自己决定，写进aria2.conf就行

我还是用gedit创建

sudo  gedit  /etc/aria2/aria2.session

保存就可以生成了。

然后就基本配置完毕了

启动aria2试试

aria2c  --conf-path=/etc/aria2/aria2.conf  -D

可根据自己的aria2.conf路径修改--conf-path=的参数

运行成功即可使用浏览器的aria2插件了。

建议将运行命令设为开机启动
