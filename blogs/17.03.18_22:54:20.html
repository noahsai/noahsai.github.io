deepin在登陆界面处出现死循环且无法进入tty的解决办法

17.03.18_22:54:20

这个问题的具体症状是本应该出现登陆界面的时候出现黑屏+光标/鼠标，

但是就是没有登陆界面，而且每一两秒就屏幕就闪一次，一直循环，tty也按不出来，一直强制跳回黑屏界面。

这种情况多数都是因为某些原因lightdm起不来，系统却一直在尝试。这里要赞一下Mint，他会在尝试几次之后帮你进入tty，十分贴心。

不明白上面在说什么的可以在deepin上卸载startdde这个依赖，然后注销就可以看到了。

下面说一下解决办法吧。

以下例子是用于解决卸载startdde后出现的问题，触类旁通，不是这个问题的自己随机应变，别照抄。

重启电脑，在高级选项里进入recovery mode

选择 root shell

然后需要讲文件系统挂载为读写状态，百度得知

mount -o remount,rw / 

接着卸载lightdm（或者将导致问题的依赖卸载掉，例如是安装了显卡驱动才造成的，那么直接卸载显卡驱动就好了，就不用动lightdm了）

apt-get purge lightdm 

重启

reboot 

重启后应该会自动进入tty模式（假如是显卡驱动造成的，上一步卸载显卡驱动后重启，应该就可以看到登陆界面了）

登陆tty后

将之前卸载掉的startdde和lightdm安装回去

sudo apt-get install lightdm startdde 

最后重启

sudo reboot 


------------------------------------【注意】--------------------------------

①root  shell里是没联网的，所以只能通过卸载解决问题。

②进入recovery mode里有enable network的选项的，但是我没试过，或者能启动网络功能，至于会不会自动连上网络我就不知道了，因为我除了root shell能正常使用之外，其他功能都不行～～
